Stackdust

See where the gigabytes went. Sweep the dust your stack leaves behind.

A macOS disk-space analyzer that knows dev junk when it sees it — DerivedData, node_modules, package caches, simulators — and only ever moves things to the Trash.

Download for macOS brew install stackdust
Free & open source · macOS 15+ · Apple silicon & Intel · GitHub
Why Stackdust

Your disk fills itself. Now you can see how.

One scan draws your whole disk as a sunburst — every ring a directory level, every arc sized by what it actually takes on disk. Hard links and firmlinks counted once, iCloud-evicted folders left in the cloud where they belong.

🞄

Built for dev junk

Xcode DerivedData and device support, simulators, node_modules, Rust targets, Gradle, Docker VM disks, package-manager caches, app caches, logs — found and grouped by category.

🗑

Trash, never rm

Cleanup means move to Trash — always recoverable, never a permanent unlink. Archives and device backups that can't be regenerated are never touched without you.

⚖︎

Honest about risk

Every reclaimable item carries a tier: safe to regenerate, costs_time to re-download, or loses_state forever. You decide where the line is.

⌘

In-app updates

Check for updates from the app menu; new versions install themselves via Sparkle. Or stay on Homebrew — both channels ship the same build.

For agents

A CLI your coding agent already knows how to use.

Claude, Codex, or a cron job — stackdust treats programs as first-class users: JSON everywhere, stable exit codes, never interactive, and a dry-run-by-default cleanup that physically cannot delete permanently.

❯ stackdust dev ~ --json
{"items": [
  {"path": "~/Library/Developer/Xcode/DerivedData", "category": "xcodeBuild", "risk": "safe", "bytes": 38214592512},
  {"path": "~/Library/Developer/CoreSimulator/Devices", "category": "simulators", "risk": "loses_state", "bytes": 21474836480},
  …
], "total_bytes": 96636764160}

❯ stackdust clean ~ --category xcodeBuild,packageCache --min-size 1G --yes
Moved 14 items (61.3 GB) to the Trash.
safe — regenerates at build time costs_time — comes back on demand loses_state — asks a human first
Install

Two ways in.

Homebrew recommended

brew tap thoughtf00l/tap
brew install --cask stackdust

Installs Stackdust.app to /Applications. Updates arrive in-app, or via brew upgrade --greedy. The stackdust CLI builds from source with swift build.

Direct download

Stackdust.dmg → drag to Applications

Stackdust isn't notarized with Apple yet, so the first launch needs a blessing: open System Settings → Privacy & Security and click Open Anyway. After that, updates arrive in-app.
